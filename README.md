# Building Updates

A digital signage dashboard for residential buildings. Displays real-time service status, upcoming events, and scrolling advisories on lobby screens.

## Deploy

Render reads `render.yaml` and auto-provisions a PostgreSQL database and web service. The app seeds itself with demo data on first start. Free tier available (DB expires after 90 days).

### Admin Setup

The first time you visit `/admin`, you'll be prompted to create your admin account (username + password). This only appears when no users exist. `COOKIE_SECRET` is auto-generated by the Render blueprint.

There are three roles: **VIEWER** (read-only, for lobby screens), **EDITOR** (can manage content), and **ADMIN** (can manage users and content). The first account is always ADMIN.

<details>
<summary><strong>Full setup (enables auto-deploy)</strong></summary>

1. **Fork this repo** to create your own copy
2. **Connect GitHub to Render** — Go to [Render Dashboard](https://dashboard.render.com) and connect your GitHub account under Account Settings if you haven't already
3. **Create a Blueprint** — Go to [New Blueprint](https://dashboard.render.com/select-repo?type=blueprint) and select your forked repo from the list (don't paste a URL)
4. **Done** — Every push to `main` auto-tags and advances `deploy/staging`. Use `./promote.sh` to deploy a tag to prod

</details>

## Local Development

Requires Node.js 20+ and Docker.

```bash
# Start Postgres
docker compose up -d

# Install deps and set up database
npm install
npx prisma db push

# Run
npm run dev
```

- **Dashboard** (lobby display): http://localhost:3000
- **Admin panel**: http://localhost:3000/admin

To stop Postgres: `docker compose down` (data persists in the Docker volume).

## Project Structure

```
server/
  app.ts              # Express app (routes, middleware, CSRF, role guards)
  index.ts            # Server entry point (seeding, startup)
  db.ts               # Prisma client
  sse.ts              # Server-sent events for real-time updates (global + named channels)
  constants.ts        # Server-side defaults (scroll speeds)
  routes/             # REST API endpoints (services, events, advisories, config, snapshots, auth, users)
  middleware/         # errorHandler (400/404/500), auth (401/403, roles), csrf
  utils/              # CRUD route factory, storage abstraction, Prisma session store
src/
  pages/
    Dashboard.tsx     # Lobby display (auto-scroll, real-time)
    Admin.tsx         # Content management UI
    Login.tsx         # Authentication page
  components/         # Header, ServiceTable, EventCard, AdvisoryTicker, AutoScrollCards, ProtectedRoute
  components/admin/   # Section editors, ImagePicker, MarkdownEditor, SnapshotHistory, SpeedSlider
  constants/          # Shared constants (colors, defaults, status presets)
  contexts/           # AuthContext (session state)
  styles/             # Shared style objects
  types.ts            # TypeScript interfaces
  utils/              # API client, markdown parser, timeAgo
prisma/
  schema.prisma       # Data models (Service, Event, Advisory, BuildingConfig, etc.)
tests/
  api/                # API + DB integration tests
  component/          # React component tests (jsdom)
  unit/               # Pure unit tests (apiClient, markdown, timeAgo)
scripts/              # Render build/start scripts
```

## Development

### Running Tests

```bash
npm test              # Run tests (needs Postgres running)
npm run test:watch    # Run tests in watch mode
npm run e2e:local     # Run Playwright suite locally (requires E2E_* creds)
```

### Database Changes

```bash
# Edit prisma/schema.prisma, then:
npx prisma db push
```
